# Strategic Compact

자동 compact는 작업 중간에 발생하여 중요한 컨텍스트를 잃을 수 있다.
이 스킬은 논리적 전환점에서 수동 compact를 권장한다.

## Compact 시점

### 권장
- **탐색 → 계획**: 코드 탐색 완료 후, 계획 수립 전
- **계획 → 구현**: 계획이 확정된 후, 코딩 시작 전
- **구현 → 리뷰**: 구현 완료 후, `/code-review` 전
- **디버깅 → 새 작업**: 버그 수정 완료 후, 다른 작업 전
- **실패한 접근 → 재시도**: 막다른 길 도달 후, 새 방향 전

### 금지
- 구현 중간 (변수명, 파일 경로, 상태 유실)
- 멀티 파일 수정 도중
- 테스트 실패 디버깅 중

## Compact 후에도 유지되는 것

- CLAUDE.md 지침
- TodoWrite 목록
- memory 파일 (`~/.claude/projects/*/memory/`)
- git 상태 (브랜치, 커밋 이력)
- 디스크에 저장된 파일

## Compact 시 유실되는 것

- 이전 대화 내용 (요약으로 대체)
- 읽었던 파일 내용
- 중간 추론 과정
- 도구 호출 결과

## 실천 방법

긴 세션에서 단계 전환 시:
1. 현재 진행 상황을 TodoWrite에 기록
2. 핵심 결정사항이나 발견을 메모 (필요시 memory 파일에)
3. `/compact` 실행
4. compact 후 TodoWrite 목록으로 작업 재개
