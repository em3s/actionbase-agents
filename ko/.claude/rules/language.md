# 언어 규칙 (MANDATORY)

이 규칙은 **모든 출력에 예외 없이 적용**된다. 위반 시 사용자가 수동으로 수정해야 하므로, 반드시 출력 전에 확인할 것.

## 모드 판별

`git remote get-url origin`으로 자동 감지한다:

- origin이 `*/actionbase`이고 `kakao/actionbase`가 아님 → **Fork 모드**
- 그 외 (kakao/actionbase, actionbase가 아닌 레포 등) → **Non-fork 모드**

## Non-fork 모드

| 출력 | 언어 | 예외 없음 |
|------|------|----------|
| 사용자와의 대화 | 한국어 | |
| 커밋 메시지 | **영어** | ⚠️ |
| PR 제목/본문 | **영어** | ⚠️ |
| 이슈 제목/본문 | **영어** | ⚠️ |
| 코드 리뷰 코멘트 | **영어** | ⚠️ |
| 코드, 주석 | **영어** | ⚠️ |
| 계획, 초안 제공 시 | **영어** | ⚠️ |

**핵심: 대화만 한국어. 나머지는 전부 영어.**

사용자가 한국어로 "PR 본문 초안 줘"라고 요청해도, 초안 자체는 **반드시 영어**로 작성한다.
사용자가 한국어로 "커밋해줘"라고 요청해도, 커밋 메시지는 **반드시 영어**로 작성한다.

## Fork 모드 (개인 fork 작업)

| 출력 | 언어 |
|------|------|
| 사용자와의 대화 | 한국어 |
| 커밋 메시지 | 한국어 |
| PR 제목/본문 | 한국어 |
| 이슈 제목/본문 | 한국어 |
| 코드 리뷰 코멘트 | 한국어 |
| 코드, 주석 | **영어** |
| upstream 대상 (`/patch-upstream`) | **영어** |

## 자기 검증

산출물을 출력하기 전에 반드시 확인:
1. 현재 모드가 무엇인가? (git remote origin 확인)
2. 이 산출물은 대화인가, 아티팩트인가?
3. 아티팩트라면 올바른 언어로 작성되었는가?
