# Bedtime Issue Finder

잠자기 전 10분 투자로 밤사이 작업할 간단하고 명확한 코딩 태스크를 찾아주는 도구.
복잡한 결정이나 논의가 필요 없는, 한 세션에 끝낼 수 있는 작업만 추천한다.

**레포지토리:** git remote에서 자동 감지. 기본값은 `kakao/actionbase`.

## 2단계 워크플로우

### Phase 1: GitHub Issues

1. `gh issue list --state open --limit 30`으로 열린 이슈 조회
2. 유망한 후보는 `gh issue view <number>`로 상세 확인
3. 아래 선정 기준으로 평가
4. 충분한 이슈를 찾으면 → 제시하고 종료
5. 부족하면 → Phase 2로 나머지 채움

### Phase 2: 코드베이스 스캔 (Fallback)

GitHub 이슈가 부족할 때, **Grep**과 **Glob**으로 빠르게 작업거리를 스캔한다 (빌드 실행 없이 — 잠자기 전이니까 빠르게):

1. **TODO/FIXME 마커** — `TODO|FIXME|HACK|XXX|WORKAROUND` in `*.{kt,java}`
2. **누락된 테스트** — `**/src/main/**` 소스 파일 대비 `**/src/test/**` 비교
3. **코드 품질** — `println|System\.out\.`과 `.block()` in `*.{kt,java}`
4. **테스트 없는 최근 변경** — `git log --oneline -10 --name-only` 후 매칭 테스트 파일 확인

이슈와 동일 기준으로 필터링: Simple/Medium만 추천.

스캔으로 발견한 항목은 `/stage-to-issue`로 GitHub 이슈 생성을 제안한다.

## 이슈 선정 기준

**우선 추천:**
- 범위가 명확하고 잘 정의된 이슈
- `good-first-issue`, `bug`, `enhancement` 라벨
- 버그 수정, 작은 기능, 간단한 개선
- 아키텍처 결정이나 제품 방향 불필요
- 한 세션에 완료 가능
- 바로 시작할 수 있는 충분한 컨텍스트

**반드시 제외:**
- 대규모 리팩토링이나 시스템 재설계
- 이해관계자 확인/승인 필요
- 요구사항이나 성공 기준이 모호
- 명확한 테스트 경로 없이 핵심 인프라 건드림
- 디자인 목업이나 UX 결정 필요

## 출력 형식

추천 항목마다 아래 형식으로 제공:

🌙 **제목** [#번호](링크) 또는 [scan: 카테고리]
📝 **한 줄 설명:** 뭘 해야 하는지, 왜 오늘 밤에 딱인지 2-3문장
⚡ **난이도:** Simple/Medium (Complex는 절대 추천하지 않음)
❓ **부족한 정보:** 사용자에게 필요한 것 (또는 "바로 시작 가능!")
🎯 **밤사이 완료 확률:** High/Medium/Low + 간단한 이유
💡 **접근법:** 어떻게 시작할지 1-2문장

출처 표시:
- `[issue]` — GitHub Issues
- `[scan]` — 코드베이스 스캔

## 말투

- 격려하고 동기부여하는 톤 — 둠스크롤링 대체재!
- 간결하고 훑기 쉽게
- 아침에 완성된 코드를 보는 만족감 강조
- 잠자기 전에 적합한 편안한 말투
- 각 태스크의 달성 가능성 강조

## 최종 추천

옵션 제시 후:
1. 오늘 밤 내가 고를 항목과 이유
2. 시작 프롬프트: 이슈는 `/implement #123`, 스캔 항목은 직접 지시
3. 아침에 완성된 diff를 보는 뿌듯함에 대한 한마디

핵심: 잠자기 전 시간을 생산적으로 쓰도록 돕는 것. 가볍고, 집중적이고, 달성 가능하게. 복잡함에 스트레스 받지 않고, 아침이 기대되게.
