# Continue 명령어

기존 PR에서 `/implement` 워크플로우를 재개합니다.

## 사용법

- `/continue 42`
- `/continue https://github.com/kakao/actionbase/pull/42`

## 에이전트 서명

모든 업데이트에 CLI와 모델을 포함하세요.
형식: `<cli> (<model>)` -- 예: `claude code (opus 4.6)`, `opencode (codex 5.2)`

PR을 시작한 에이전트와 다른 에이전트일 수 있습니다. 자신의 작업에 서명하고 이전 항목을 보존하세요.

## 단계

1. **PR 조회** -- `gh pr view <arg>` (본문, 상태, 브랜치)
2. **브랜치 체크아웃** -- `gh pr checkout <arg>`
3. **코멘트 조회** -- `gh pr view <arg> --comments`로 리뷰 이력 확인
4. **이슈 조회** -- PR 본문에서 `Closes #N` 추출, `gh issue view <N>`
5. **진행 상황 파악** -- 완료된 `[x]`와 미완료 `[ ]` 항목, 마지막 리뷰 라운드 번호, 작업한 에이전트 확인
6. **상태 제시** -- 완료된 항목, 담당 에이전트, 다음 작업을 사용자에게 표시
7. **중단** -- 사용자 확인을 기다림
8. **재개** -- 다음 미완료 항목부터 `/implement` 5-10단계를 따라 진행하며 완료 항목에 서명

## 규칙

- 사용자 확인 없이 절대 진행하지 않음
- 모든 `/implement` 규칙 준수 (에이전트 서명 및 이력 보존 포함)
- 중단된 지점부터 리뷰 라운드 번호 계속 사용
- **작업에 서명** -- 단계 완료, 리뷰 코멘트, PR 본문 업데이트
- **이력 보존** -- 다른 에이전트의 항목을 절대 삭제하지 않음
- 모든 출력은 한국어로 작성 (코드와 코드 내 주석은 영어)
